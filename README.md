# Pi Documentation (Bilingual: English + 简体中文)

A self-hosted VitePress static site that republishes the open Markdown docs from the
upstream Pi project (`github.com/earendil-works/pi`, path `packages/coding-agent/docs/`).
English is the default language; a Simplified Chinese translation is generated by a
local LLM and served at `/zh/`.

**Live site:** <https://wangxiancao.github.io/pi_doc_zh/>
(中文: <https://wangxiancao.github.io/pi_doc_zh/zh/>)

This template does not reproduce the official `pi.dev` website — only the open docs
are rebuilt.

## What This Repo Contains

- Three-layer content pipeline: `content/` (source) → `docs/` (build tree) → `dist/`
- VitePress config with i18n locales in `.vitepress/config.ts`
- Locale routing single-source in `locales.json`
- Scripts: `sync-pi-docs`, `prepare-docs`, `generate-sidebar`, `translate-locale`,
  `validate-i18n`
- Translation providers: `scripts/providers/local-command.mjs` (local Qwen),
  `scripts/providers/deepseek-api.mjs` (remote fallback)
- GitHub Pages workflow in `.github/workflows/pages.yml`
- Local static server helper in `site-server.sh`

## Requirements

- Node.js 20 or newer
- npm
- A local checkout of `https://github.com/earendil-works/pi` (only for syncing upstream)
- A GitHub repository where this folder is pushed
- (For translation only) either the local Qwen supervisor running, or a `DEEPSEEK_API_KEY`

## Architecture: Three Layers

```text
content/   <- persistent source, tracked in git (en + zh + ...)
docs/      <- VitePress build tree, derived, gitignored, rebuilt every build
dist/      <- GitHub Pages artifact, gitignored
```

Edit `content/`, never `docs/`. `npm run build` regenerates `docs/` from `content/`
automatically (via the `prebuild` hook: `prepare-docs && generate-sidebar`).

## Quick Start

```bash
npm install
npm run sync -- /path/to/pi     # upstream -> content/en/  (path = Pi repo ROOT)
npm run build                   # prebuild: prepare-docs + generate-sidebar
```

Example if the Pi repository is cloned next to this folder:

```bash
npm run sync -- ../pi
npm run build
```

The generated site is written to `dist/`. Preview locally:

```bash
SITE_BASE=/ npm run build && ./site-server.sh start
```

During development:

```bash
npm run dev      # VitePress dev server on 127.0.0.1 (auto-runs prepare-docs + generate-sidebar)
```

The development server binds to `127.0.0.1`; do not expose it publicly.

## Translation (English → <locale>)

Translations live in `content/<locale>/` and are generated by `translate-locale.mjs`
with a pluggable provider:

```bash
# Default: local Qwen3.6-35B-A3B via qwen36-supervisor (start it first)
/root/soft/llama.cpp/qwen36-supervisor &
npm run translate -- zh

# Remote fallback
TRANSLATION_PROVIDER=deepseek-api DEEPSEEK_API_KEY=... npm run translate -- zh
```

Translation is incremental (sha256 cache, skips unchanged files) and preserves code
blocks, URLs, frontmatter, link targets, and heading anchors. After translating:

```bash
npm run build && npm run validate-i18n
```

Add a new language: add an entry to `locales.json`, create `content/<dir>/index.md` +
`docs.json`, then `npm run translate -- <dir>`.

## GitHub Pages Deployment

1. Push this folder as its own GitHub repository.
2. `Settings -> Pages -> Build and deployment -> Source` = `GitHub Actions`.
3. Push to `main`; wait for the `Deploy Pi docs` workflow.

The workflow runs `npm ci → npm run build → npm run validate-i18n`, uploads `dist/`,
and deploys via GitHub's official Pages artifact flow.

## Base URL Rules

Auto-detected from the repo name: `USERNAME/pi_doc` → base `/pi_doc/`;
`USERNAME.github.io` → base `/`. Override with the `SITE_BASE` repository variable
(`SITE_BASE=/` for custom domains). This repo's base auto-resolves to `/pi_doc_zh/`.

## Updating From Upstream Pi

```bash
git clone https://github.com/earendil-works/pi.git
npm run sync -- ../pi              # -> content/en/
npm run build && npm run validate-i18n
# then re-translate locales whose source changed:
npm run translate -- zh
npm run build && npm run validate-i18n
# commit content/en/ + content/zh/ changes; push
```

## Important Boundary

This template publishes documentation content. It does not include the official
`pi.dev` homepage, visual design, news/packages/models pages, or search implementation
— those are outside the public Pi repository.

## Troubleshooting

- **Links broken on Pages**: check `SITE_BASE`.
- **404 right after enabling Pages**: GitHub may add a starter `static.yml` ("Deploy
  static content to Pages") that deploys the repo root with no build step; `dist/` is
  gitignored so the root has no `index.html`. Delete `.github/workflows/static.yml`,
  keep only `pages.yml`, push.
- **Edits to `docs/` vanish**: edit `content/` instead; `docs/` is derived.
- **Build fails on "id attribute X is not unique"**: duplicate heading slug — re-run
  translation (`translate-locale` dedupes slugs like `when-it-triggers` → `-1`).
- **`validate-i18n` reports missing internal link**: auto-skipped if also broken in
  `content/en/` (upstream dead link); otherwise fix the translation.
- **Translate fails "qwen36-supervisor 未启动"**: start the supervisor first.
- **Sidebar missing**: run `npm run generate-sidebar`.
- **Sync fails**: confirm the path contains `packages/coding-agent/docs/docs.json`.
- **Pages does not deploy**: confirm Source = `GitHub Actions`, Actions enabled, and
  the workflow has `pages: write` + `id-token: write`.
- **Deploy job fails but build job succeeds**: `actions/deploy-pages` 偶发 under
  `concurrency: pages`; Re-run failed jobs in the Actions UI.

## Dependency Audit Note

`npm audit --omit=dev` should report no production vulnerabilities. VitePress is a
build-time dependency. If full `npm audit` reports a Vite/esbuild dev-server advisory,
do not expose `npm run dev` beyond localhost. The published site is static HTML/CSS/JS
under `dist/`.
